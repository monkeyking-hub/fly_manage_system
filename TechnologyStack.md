# 技术栈要求文档

## 团队协作

- **版本控制**：使用 Git 进行版本管理，所有团队成员应熟练掌握 Git 的基本操作，包括 `clone`、`commit`、`push`、`pull`、`merge` 等命令。
- **项目管理工具**：使用 Jira 或 Trello 等项目管理工具进行任务跟踪和进度管理。
- **代码质量与审查**：所有提交的代码需经过代码审查，使用静态代码分析工具（如 SonarQube）确保代码质量。
- **文档**：开发文档和 API 文档需及时更新，使用 Markdown 格式进行编写并托管到 GitHub 或 Gittee 上。

## 前端开发
- **需学习技术** QT设计师界面,QSS/CSS样式表,响应式设计界面
- **框架**：前端界面使用 Qt 进行开发，基于 QtWidgets 或 QtQuick 实现交互式 GUI。
- **UI 设计工具**：使用 Qt Designer 设计界面布局，界面设计与开发工作分离，方便协作。
- **前端架构**：采用 MVC（Model-View-Controller）架构设计前端部分，界面部分通过信号与槽机制与后台交互。
- **前端样式**：通过 Qt 样式表（QSS）对 UI 进行定制化样式设计，保持界面的一致性。
- **响应式设计**：界面应具备一定的响应式设计，支持多种分辨率和窗口尺寸。

## 后端开发
- **需学习技术** http服务器基础学习,多线程并发编程,数据库orm思想,redis非关系型数据库,日志系统

- **数据库**：使用 MySQL 进行数据存储，采用 ORM进行数据库操作，确保数据库操作的抽象和安全。
- **API**：使用 RESTful API 设计，后端通过 HTTP 提供接口与前端进行数据交换。
- **安全性**：后端系统需要保证用户数据的安全，采用 HTTPS 加密传输，并使用 OAuth 或 JWT 进行用户认证和授权。
- **缓存机制**：对于高频查询，使用缓存策略（如 Redis）提升性能，减少数据库负担。
- **日志记录**：使用 Qt 的日志记录功能，结合日志分析工具（如 ELK Stack）进行系统监控。

## 数据库设计

- **数据库模型**：根据业务需求设计数据模型，包括用户、航班、订单等表格。
- **事务管理**：保证数据一致性和完整性，对于涉及多个表格的操作使用事务处理。
- **索引优化**：为查询频繁的字段设计适当的索引，以提升查询效率。

## 部署与运维

- **部署方式**：根据团队需求，选择合适的部署方式（如 Docker 部署、虚拟机、云平台等）。
- **容器化**：考虑使用 Docker 容器化应用，方便在不同环境中快速部署。
- **自动化部署**：使用 CI/CD 工具（如 Jenkins、GitLab CI）进行自动化部署，提高开发效率和系统稳定性。
- **监控与告警**：使用监控工具（如 Prometheus）监控系统健康状态，并配置告警策略。

## 质量保证与测试

- **单元测试**：Catch2：另一个 C++ 的单元测试框架
- **集成测试**：Postman：广泛使用的 API 测试工具
- **UI 测试**：Qt Test：能够进行自动化测试，尤其适合 Qt 项目。。
- **性能测试**：JMeter：Apache JMeter 是一种广泛使用的性能测试工具
- **安全测试**：OWASP ZAP (Zed Attack Proxy)：一款免费的开源 Web 安全扫描工具
