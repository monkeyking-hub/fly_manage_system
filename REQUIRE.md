# 航班票务管理系统需求分析

## 1. 系统概述

本系统是一个航班票务管理系统，旨在为用户提供在线查询、预订航班、查看和管理订单，以及完成支付等功能。用户可以通过注册和登录来使用系统，查询航班信息，预订航班并进行支付，系统会为每个用户提供订单管理和支付状态更新等功能。

## 2. 功能模块

### 2.1 用户模块
用户模块提供了用户注册、登录、个人信息管理等功能。
#### 功能列表：
- **用户注册**：用户通过提供用户名、密码、邮箱、手机等信息进行注册。
- **用户登录**：已注册的用户通过用户名和密码进行登录。
- **用户信息管理**：用户可以修改自己的个人信息，如修改密码、更新联系方式等。
- **用户注销**：用户可以注销自己的账户。

### 2.2 航班管理模块
航班管理模块用于展示和查询航班信息。
#### 功能列表：
- **航班查询**：用户可以根据出发地、目的地、日期等条件搜索航班。
- **航班展示**：显示符合条件的航班信息，包括航班号、起飞和降落时间、出发地、目的地、座位余量、票价等。
- **航班选择**：用户可以选择想要预订的航班，进入订单创建页面。

### 2.3 订单管理模块
订单管理模块提供用户查询和管理自己的航班订单。
#### 功能列表：
- **订单创建**：用户在选择航班后，可以创建订单，选择座位，确认航班信息和票价，提交订单。
- **订单查询**：用户可以查看自己的订单记录，包括已预订、已取消、已完成等状态。
- **订单取消**：用户可以在未支付前取消订单，系统会删除该订单。
- **订单详情**：用户可以查看订单的详细信息，包括航班、价格、状态等。

### 2.4 支付模块
支付模块提供支付功能，并能够更新支付状态。
#### 功能列表：
- **选择支付方式**：用户选择支付方式（如信用卡、支付宝等）。
- **支付流程**：用户确认支付后，系统处理支付，并返回支付结果（成功或失败）。
- **支付状态更新**：系统更新支付状态，并通知用户支付是否成功。

## 3. 系统非功能性需求

### 3.1 系统性能
- 支持每秒钟至少处理 1000 个用户请求。
- 系统响应时间不超过 3 秒。

### 3.2 安全性
- 密码使用加密存储，避免明文存储。
- 支付模块应使用安全的支付接口，并保证用户支付信息的安全。
- 用户信息在传输过程中应使用 HTTPS 加密。

### 3.3 可用性
- 用户界面需要简洁、直观，能够快速引导用户完成操作。
- 系统需要进行充分的错误处理和用户提示，确保系统稳定运行。

### 3.4 扩展性
- 系统应具备良好的扩展性，能够在未来添加更多功能（如多种支付方式、航班筛选条件等）。
- 支持在未来增加不同地区的航班和支付接口。

## 4. 系统架构

### 4.1 前端（客户端）
- **Qt Widgets** 或 **Qt Quick (QML)** 用于开发用户界面。
  - 通过 **Qt Designer** 创建各个界面（如登录界面、航班查询界面、订单管理界面等）。
  - **Qt SQL** 用于与 MySQL 数据库进行交互。

### 4.2 后端（服务器端）
- **MySQL** 数据库：存储用户信息、航班信息、订单信息和支付信息。
- **Qt SQL**：Qt 提供的 SQL 连接模块，用于执行数据库查询和更新。

### 4.3 数据库设计
#### 4.3.1 数据表设计
- **users**：存储用户基本信息。
- **flights**：存储航班信息。
- **orders**：存储用户的航班预订订单信息。
- **payments**：存储用户支付信息。
